# Notebook Guide

## Notebook Mapping to Assignment

- `Task1_Product_Recommendation.ipynb`
  - Data cleaning, merge, EDA, model training, metrics, and model persistence.

- `Task2_Face_Image_Processing.ipynb`
  - Image collection workflow, augmentation, and feature extraction.

- `Task3_Voice_Audio_Processing.ipynb`
  - Audio collection workflow, waveform/spectrogram checks, augmentation, and feature extraction.

- `Task4_Multimodal_Integration.ipynb`
  - End-to-end gate logic demonstration: face -> voice -> product prediction.

## Why these notebook names?

The `Task1`..`Task4` naming is intentional:
- Aligns directly with assignment sections.
- Easier for markers to follow.
- Avoids confusion from older template-style names.

## Execution policy

Only executed notebooks are kept for submission.

Each notebook should:
- Run top-to-bottom without errors.
- Include short explanation cells before/after code outputs.
- Save required artifacts and plots.
