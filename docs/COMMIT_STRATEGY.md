# Commit Strategy (10+ Commits)

Use this sequence to keep your git history clean and markable.

1. `chore: initialize repository and baseline structure`
2. `feat: add raw-to-processed data pipeline`
3. `feat: add expanded EDA visualizations`
4. `feat: implement multi-model training and evaluation`
5. `feat: add confusion matrix and feature importance plots`
6. `feat: rebuild Task1 product notebook with richer explanations`
7. `feat: rebuild Task2 face notebook with augmentation and output notes`
8. `feat: rebuild Task3 voice notebook with augmentation and output notes`
9. `feat: rebuild Task4 multimodal integration notebook`
10. `docs: add teammate quickstart and assignment checklist`
11. `docs: update README and notebook naming`
12. `chore: clean redundant notebooks and helper files`

You can squash later only if your course rules allow it.
